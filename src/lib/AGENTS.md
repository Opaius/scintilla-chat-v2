## Purpose

Remult + Better Auth integration: server API config, entities, D1 database, Svelte 5 reactivity.

## Ownership

- `server/api.ts` — Remult API config with D1/fallback dataProvider
- `server/auth.ts` — Better Auth instance (email/password), connected via `@nerdfolio/remult-better-auth`
- `server/data-provider.ts` — Resolves D1 dataProvider on first request for Better Auth adapter
- `remult.ts` — Svelte 5 reactivity for remult.auth
- `entities/` — Remult entity definitions (including `entities/auth.ts` generated by Better Auth CLI)
- `entities/auth.ts` — Auto-generated by `@better-auth/cli generate`, do not edit directly

## Local Contracts

- API config at `$lib/server/api.ts`, re-exported via `$lib/server/api`
- Better Auth config at `$lib/server/auth.ts`, routes at `src/routes/api/auth/[...auth]/+server.ts`
- Route catch-all at `src/routes/api/[...remult]/+server.ts` uses `GET, POST, PUT, DELETE` destructured from `api`
- D1 dataProvider set per-request via `initRequest` from `event.platform.env.DB`; also resolved once for Better Auth via `data-provider.ts`
- Falls back to Remult default `JsonFileDataProvider('./db')` when no D1 binding
- Remult `getUser` delegates to Better Auth `auth.api.getSession()` via lazy dynamic import to avoid circular deps

## Work Guidance

- Register new entities in `server/api.ts` `remultApi({ entities: [...] })`
- Regenerate auth entities after changing Better Auth plugins: `bunx better-auth generate --config src/lib/server/auth.ts --output src/lib/entities/auth.ts`
- Better Auth schema tables match entity key names (`user`, `session`, `account`, `verification`)
- `BETTER_AUTH_URL` env var sets the base URL (defaults to `http://localhost:5173`)
